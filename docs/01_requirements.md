# 要件定義書

## 1. システム概要

### 1.1 システム名

PicoProg（学習記録共有アプリ）

### 1.2 目的

職業訓練校でPythonやデータベースを学習している生徒が、日々の学習内容や制作物をつぶやき形式で記録・共有し、お互いの学習状況を確認できるSNS型アプリケーション。

### 1.3 対象ユーザー

- 職業訓練校の生徒
- 同じクラスの仲間との学習記録共有を目的とする

---

## 2. 機能要件

### 2.1 ユーザー管理機能

#### 2.1.1 新規登録機能

- ユーザー名、メールアドレス、パスワード（確認用含む）を入力して新規登録できる
- 登録後、自動的にログイン状態となりタイムライン画面へ遷移する

#### 2.1.2 ログイン機能

- メールアドレスとパスワードでログインできる
- ログイン成功後、タイムライン画面へ遷移する

#### 2.1.3 ログアウト機能

- ログアウトボタンからログアウトできる
- ログアウト後、ログイン画面へ遷移する

#### 2.1.4 ユーザー設定機能

- ユーザー名の変更ができる
- パスワードの変更ができる（現在のパスワードによる本人確認あり）
- アイコン画像のアップロードができる
- プロフィール情報（自己紹介文）の変更ができる

### 2.2 投稿管理機能

#### 2.2.1 新規投稿機能

- 投稿内容（テキスト）を入力できる
- カテゴリを複数選択できる（学習中、制作中、Python基礎、データベース等）
- 投稿ボタンで投稿を完了し、タイムライン画面へ遷移する

#### 2.2.2 投稿編集機能

- 自分の投稿内容を編集できる
- カテゴリの変更ができる
- 編集後、タイムライン画面へ遷移する

#### 2.2.3 投稿削除機能

- 自分の投稿を削除できる
- 削除前に確認モーダルを表示する

### 2.3 タイムライン機能

#### 2.3.1 投稿一覧表示機能

- すべてのユーザーの投稿を新しい順に表示する
- 1ページあたり10件表示し、ページネーションで切り替えられる
- 各投稿には以下の情報を表示する
  - ユーザー名・アイコン
  - 投稿日時
  - 投稿内容
  - カテゴリタグ
  - いいねボタン（カウント表示）
  - お気に入りボタン（カウント表示）
  - コメントボタン
  - 編集・削除ボタン（自分の投稿のみ）

#### 2.3.2 検索機能

- キーワードで投稿を検索できる
- カテゴリで絞り込みができる

### 2.4 いいね機能

- 投稿にいいねを付けることができる
- 既にいいね済みの投稿は、再度クリックで解除できる
- いいね時に投稿者へ通知を送信する

### 2.5 お気に入り機能

#### 2.5.1 お気に入り登録機能

- 投稿にお気に入りを付けることができる
- 既にお気に入り済みの投稿は、再度クリックで解除できる
- お気に入り時に投稿者へ通知を送信する

#### 2.5.2 お気に入り一覧表示機能

- 自分がお気に入りした投稿の一覧を表示できる
- 表示内容はタイムラインと同様
- 1ページあたり10件表示し、ページネーションで切り替えられる

### 2.6 コメント機能

#### 2.6.1 コメント投稿機能

- 投稿に対してコメントを書き込むことができる
- コメント時に投稿者へ通知を送信する

#### 2.6.2 コメント表示機能

- 投稿に対するコメントを一覧表示できる
- コメントには投稿者名・アイコンと投稿日時を表示する

#### 2.6.3 コメント削除機能

- 自分のコメントを削除できる
- 削除前に確認モーダルを表示する

### 2.7 通知機能

- いいね・コメント・お気に入りが付いた際に、投稿者へ通知を生成する
- ナビゲーションに未読通知件数のバッジを表示する
- 通知一覧を表示できる（1ページあたり10件）
- 通知カードをクリックすると既読にしつつ対象投稿へ遷移する
- 既読から30日後に通知を自動削除する

### 2.8 プロフィール機能

- ユーザーのプロフィール（アイコン・自己紹介文）と投稿一覧を表示できる

### 2.9 管理者機能

- 管理者権限を持つユーザーのみアクセスできる管理パネルを提供する
- ユーザー一覧を表示・削除できる
- 全ユーザーの投稿をキーワード検索・一覧表示・削除できる
- 削除前に確認モーダルを表示する

## 3. 非機能要件

### 3.1 パフォーマンス

- ページ遷移は3秒以内に完了すること

### 3.2 セキュリティ

- パスワードはハッシュ化して保存すること
- 全フォームにCSRFトークンを付与すること
- ログインしていないユーザーは、ログイン・新規登録画面以外にアクセスできないこと
- 投稿・コメントの編集・削除は本人のみ実行可能とすること
- 管理者機能は管理者権限を持つユーザーのみアクセス可能とすること

### 3.3 バリデーション

- サーバーサイドで必ず入力チェックを実施し、不正なリクエストからDBを保護する

| 対象 | 条件 |
|---|---|
| ユーザー名 | 3〜20文字 |
| メールアドレス | 正規表現による形式チェック |
| パスワード | 8文字以上・英字含む・数字含む |
| 投稿本文 | 1〜1000文字 |
| プロフィール | 200文字以内 |

- クライアントサイドでもリアルタイムにチェックし、エラー時はフィールドを赤くしてメッセージを直下に表示する
- 投稿フォームには文字数カウンターを常時表示する

### 3.4 使いやすさ

- シンプルで直感的なUI/UXを提供すること
- モバイル端末でも閲覧可能なレスポンシブデザインとすること

## 5. 使用技術

- バックエンド: Python (Flask)
- データベース: SQLite（sqlite3）
- フロントエンド: HTML, CSS, JavaScript
- 認証: bcrypt（パスワードハッシュ化）
- セキュリティ: Flask-WTF（CSRFトークン）
- 本番環境: PythonAnywhere

---

## 6. 制約事項

- 職業訓練校のクラス内での利用を想定
- インターネット接続環境が必要
- 画像アップロードはアイコンのみ対応
