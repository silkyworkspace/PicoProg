# 基本設計書 - PicoProg（学習記録共有アプリ）

## 1. 画面一覧 ※詳細は画面遷移図

| 画面ID | 画面名 | 主な機能 |
|---|---|---|
| S01 | ログイン画面 | メールアドレスとパスワードでログイン |
| S02 | 新規登録画面 | 新規アカウント作成 |
| S03 | タイムライン画面 | 投稿一覧表示・キーワード検索・カテゴリ絞り込み |
| S04 | 新規投稿画面 | カテゴリ選択して投稿作成 |
| S05 | 投稿編集画面 | 自分の投稿を編集 |
| S06 | コメント画面 | 投稿へのコメント表示・投稿 |
| S07 | お気に入り一覧画面 | お気に入りした投稿一覧 |
| S08 | 通知画面 | 通知一覧表示・既読化 |
| S09 | プロフィール画面 | ユーザーの投稿一覧・プロフィール表示 |
| S10 | ユーザー設定画面 | ユーザー名・パスワード・アイコン・プロフィール編集 |
| S11 | 管理者パネル画面 | ユーザー管理・全投稿管理（管理者専用） |
| S12 | エラー画面 | 404 / 500 エラー表示 |

## 2. 画面詳細

### S01　ログイン画面

| 項目 | 内容 |
|---|---|
| 入力項目 | ・メールアドレス（必須）<br>・パスワード（必須） |
| ボタン | ・ログインボタン → 認証してタイムライン画面へ<br>・新規登録リンク → 新規登録画面へ |
| バリデーション | ・未入力チェック<br>・メールアドレス形式チェック<br>・認証失敗時：「メールアドレスまたはパスワードが正しくありません」 |

### S02　新規登録画面

| 項目 | 内容 |
|---|---|
| 入力項目 | ・ユーザー名（必須、3〜20文字）<br>・メールアドレス（必須、重複不可）<br>・パスワード（必須、8文字以上・英字含む・数字含む）<br>・パスワード確認（必須、パスワードと一致） |
| ボタン | ・登録してログインボタン → 登録後、自動ログインしてタイムライン画面へ<br>・ログインリンク → ログイン画面へ |
| バリデーション | ・各項目の未入力チェック<br>・文字数チェック<br>・パスワード強度チェック（英字・数字を含む）<br>・メールアドレス重複チェック<br>・パスワード一致チェック |

### S03　タイムライン画面（ホーム）

| 項目 | 内容 |
|---|---|
| 表示内容 | ・全ユーザーの投稿を新着順に表示（10件／ページ）<br>・各投稿：ユーザーアイコン・ユーザー名、投稿日時、投稿内容、カテゴリタグ、いいねボタン（カウント）、お気に入りボタン（カウント）、コメントボタン<br>・自分の投稿には編集・削除ボタンを表示 |
| 機能 | ・キーワード検索：投稿内容をキーワードで絞り込み<br>・カテゴリ絞り込み：チェックボックスで選択<br>・いいね：ハートアイコンクリックで登録／解除<br>・お気に入り：ブックマークアイコンクリックで登録／解除<br>・ページネーション：前後ページへの切り替え |
| ナビゲーション | タイムライン / 投稿する / お気に入り / 通知（未読バッジ）/ プロフィール / 設定 / ログアウト |

### S04　新規投稿画面

| 項目 | 内容 |
|---|---|
| 入力項目 | ・カテゴリ選択（必須、複数選択可）<br>　学習段階：学習中 / 制作中 / 完成・達成<br>　技術領域：Python基礎 / Web開発(Flask) / データベース(SQL) / フロントエンド(HTML/CSS/JS) / その他<br>・投稿内容（必須、1〜1000文字、文字数カウンター表示） |
| ボタン | ・投稿ボタン → 投稿してタイムライン画面へ<br>・トップに戻る → タイムライン画面へ |
| バリデーション | ・カテゴリ未選択チェック<br>・投稿内容の未入力・文字数チェック |

### S05　投稿編集画面

| 項目 | 内容 |
|---|---|
| 表示内容 | ・既存の投稿内容とカテゴリを表示（編集可能）<br>・自分の投稿のみ編集可能（他人の投稿は権限エラー） |
| 入力項目 | 新規投稿画面と同じ |
| ボタン | ・更新ボタン → 更新してタイムライン画面へ<br>・トップに戻る → タイムライン画面へ |
| バリデーション | 新規投稿画面と同じ |

### S06　コメント画面

| 項目 | 内容 |
|---|---|
| 表示内容 | ・元の投稿（ユーザーアイコン・ユーザー名、日時、内容、カテゴリ）<br>・コメント一覧（ユーザーアイコン・ユーザー名、日時、コメント内容）<br>・自分のコメントには削除ボタンを表示 |
| 入力項目 | ・コメント内容（必須） |
| ボタン | ・コメントするボタン → コメント投稿後、同画面を再表示<br>・トップに戻る → タイムライン画面へ |
| バリデーション | ・コメント内容の未入力チェック |

### S07　お気に入り一覧画面

| 項目 | 内容 |
|---|---|
| 表示内容 | ・自分がお気に入りした投稿のみ表示（10件／ページ）<br>・表示形式はタイムライン画面と同じ |
| ボタン | ・トップに戻る → タイムライン画面へ<br>・各投稿のコメント・編集・削除ボタン |

### S08　通知画面

| 項目 | 内容 |
|---|---|
| 表示内容 | ・自分への通知一覧（いいね・コメント・お気に入り）を新着順に表示（10件／ページ）<br>・未読通知は既読通知と視覚的に区別して表示<br>・既読から30日後に自動削除される旨を表示 |
| 機能 | ・通知カードをクリック → 既読にしつつ対象投稿のコメント画面へ遷移 |
| ボタン | ・トップに戻る → タイムライン画面へ |

### S09　プロフィール画面

| 項目 | 内容 |
|---|---|
| 表示内容 | ・ユーザーアイコン・ユーザー名・自己紹介文<br>・そのユーザーの投稿一覧（新着順） |

### S10　ユーザー設定画面

| 項目 | 内容 |
|---|---|
| 入力項目 | ・ユーザー名（必須、3〜20文字）<br>・アイコン画像（任意、png／jpg／jpeg／gif、5MB以内）<br>・プロフィール（任意、200文字以内）<br>・現在のパスワード（パスワード変更時に必須）<br>・新しいパスワード（任意、8文字以上・英字含む・数字含む）<br>・新しいパスワード確認（新しいパスワードと一致） |
| ボタン | ・プロフィール更新ボタン → 更新後、設定画面を再表示<br>・パスワード変更ボタン → 変更後、設定画面を再表示<br>・トップに戻る → タイムライン画面へ |
| バリデーション | ・ユーザー名の未入力・文字数チェック<br>・画像ファイル形式・サイズチェック<br>・プロフィール文字数チェック<br>・パスワード変更時：現在のパスワード一致チェック・強度チェック・確認一致チェック |

### S11　管理者パネル画面

| 項目 | 内容 |
|---|---|
| アクセス制限 | 管理者権限（is_admin = 1）を持つユーザーのみアクセス可能。それ以外は403エラー。 |
| 表示内容 | ・ユーザー一覧（ID・ユーザー名・メールアドレス・管理者フラグ・登録日時）<br>・全投稿一覧（ID・投稿者・内容・投稿日時）、キーワード検索対応 |
| ボタン | ・ユーザー削除ボタン → 確認モーダル表示後、削除してページ再表示<br>・投稿削除ボタン → 確認モーダル表示後、削除してページ再表示 |

### S12　エラー画面

| 項目 | 内容 |
|---|---|
| 表示内容 | ・404：ページが見つからない旨とトップへ戻るリンク<br>・500：サーバーエラーの旨とトップへ戻るリンク |

## 3. 共通仕様

**認証・権限**
- ログイン必須画面：S03〜S11（未ログイン時はログイン画面へリダイレクト）
- 投稿・コメントの編集・削除：本人のみ可能（他人のリソースへのアクセスはエラー）
- 管理者パネル（S11）：管理者権限ユーザーのみアクセス可能

**メッセージ表示**
- 成功メッセージ：緑色背景のフラッシュメッセージで画面上部に表示
- エラーメッセージ：赤色背景のフラッシュメッセージで画面上部に表示

**削除確認**
- 投稿・コメント・ユーザーの削除操作はすべて確認モーダルを表示してから実行する

**セキュリティ**
- パスワードはbcryptでハッシュ化して保存
- 入力値はすべてエスケープ処理（XSS対策）
- SQLはプレースホルダーを使用（SQLインジェクション対策）
- 全フォームにCSRFトークンを設定（Flask-WTF）

**パフォーマンス**
- 投稿・お気に入り・通知の一覧は10件ごとにページネーション
- 画像アップロードは5MB以内に制限
